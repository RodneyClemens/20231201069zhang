# 百度贴吧项目 - 角色任务指令

## 1. 美工/UI前端设计Agent 任务指令

### 1.1 输入
- **页面需求**：
  - 首页：展示热门贴吧和热门帖子
  - 登录/注册页：用户认证功能
  - 贴吧列表页：分类展示贴吧
  - 贴吧详情页：帖子列表和发帖入口
  - 帖子详情页：帖子内容和评论区
  - 用户个人中心：个人信息管理

- **风格要求**：
  - 现代化设计风格，简洁易用
  - 响应式布局，适配移动端和桌面端
  - 主色调参考现有HTML原型（蓝色系为主）
  - 符合百度贴吧的整体风格和用户习惯

- **组件清单**：
  - 通用组件：按钮、输入框、卡片、导航栏、列表、模态框
  - 业务组件：贴吧卡片、帖子卡片、评论组件、用户信息卡片、富文本编辑器

### 1.2 输出
- **UI规范文档**：包含颜色、字体、间距等设计规范
- **HTML/CSS原型**：所有页面的HTML结构和CSS样式
- **组件库实现**：可复用的前端组件代码
- **交互效果实现**：基础的JavaScript交互逻辑
- **响应式设计说明**：不同屏幕尺寸的适配方案

### 1.3 格式要求
- HTML/CSS文件需按照功能模块分类存放
- 代码需包含清晰的注释
- 组件需提供使用说明
- 所有设计资源需统一整理

### 1.4 截止顺序
- **第1天**：提交UI规范文档初稿
- **第3天**：提交首页和登录/注册页原型
- **第5天**：提交贴吧相关页面原型
- **第7天**：提交组件库第一版
- **第10天**：提交所有页面原型和完整组件库

## 2. 后端程序员Agent 任务指令

### 2.1 输入
- **业务实体**：
  - User（用户）：id、用户名、密码、邮箱、手机、头像、注册时间
  - Tieba（贴吧）：id、名称、描述、创建者、创建时间、成员数
  - Post（帖子）：id、标题、内容、作者、所属贴吧、发布时间、浏览量、回复数
  - Comment（评论）：id、内容、作者、所属帖子、回复目标、发布时间
  - Category（分类）：id、名称、描述

- **接口清单**：
  - 用户认证：/api/auth/register、/api/auth/login、/api/auth/logout、/api/auth/profile
  - 贴吧管理：/api/tiebas/、/api/tiebas/{id}、/api/tiebas/{id}/follow
  - 帖子管理：/api/posts/、/api/posts/{id}、/api/posts/{id}/like
  - 评论管理：/api/comments/、/api/comments/{id}
  - 搜索功能：/api/search/、/api/search/posts、/api/search/users

- **非功能性要求**：
  - 响应时间：API接口响应时间不超过200ms
  - 安全性：密码加密存储、防SQL注入、XSS防护
  - 可扩展性：模块化设计，易于扩展
  - 兼容性：与前端技术栈兼容

### 2.2 输出
- **数据库设计文档**：包含表结构、字段说明、索引设计
- **API设计文档**：包含接口定义、请求/响应格式、示例
- **服务划分文档**：包含模块划分、组件关系
- **技术栈建议**：包含框架、库、工具的版本和选型理由
- **核心代码实现**：包含models、views、serializers等核心文件

### 2.3 格式要求
- 代码需符合Python PEP8规范
- API文档需使用RESTful风格
- 数据库设计需提供SQL建表语句
- 代码需包含单元测试
- 文档需使用Markdown格式

### 2.4 截止顺序
- **第1天**：提交数据库设计文档
- **第3天**：提交用户认证相关代码
- **第6天**：提交帖子和评论相关代码
- **第8天**：提交搜索和权限相关代码
- **第10天**：提交完整的后端服务和文档

## 3. 测试Agent 任务指令

### 3.1 输入
- **功能清单**：
  - 用户管理：注册、登录、个人信息管理、密码找回
  - 帖子管理：发布、编辑、删除、浏览、点赞
  - 评论管理：发表、回复、删除评论
  - 贴吧管理：创建、关注、浏览贴吧
  - 搜索功能：搜索帖子、用户

- **验收条件**：
  - 所有功能正常运行，无致命bug
  - 页面加载时间不超过2秒
  - 数据库查询响应时间不超过200ms
  - 系统支持100并发用户
  - 安全漏洞检测通过

### 3.2 输出
- **测试计划文档**：包含测试策略、测试范围、测试环境
- **测试用例集**：包含功能测试、性能测试、安全测试用例
- **冒烟测试清单**：核心功能快速验证用例
- **回归测试脚本**：自动化测试脚本
- **验收测试报告**：包含测试结果、问题清单、修复建议

### 3.3 格式要求
- 测试用例需包含测试步骤、预期结果
- 测试报告需包含通过率、问题严重程度分析
- 问题清单需包含优先级、复现步骤
- 文档需使用Markdown格式

### 3.4 截止顺序
- **第1天**：提交测试计划文档
- **第3天**：提交冒烟测试清单和核心功能测试用例
- **第5天**：提交完整测试用例集
- **第8天**：提交第一轮测试报告
- **第10天**：提交最终验收测试报告
