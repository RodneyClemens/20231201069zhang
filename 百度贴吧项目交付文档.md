《项目概览》：百度贴吧项目是一个类似百度贴吧的Web应用，范围包括用户管理、帖子管理、评论系统、贴吧管理和搜索功能五大核心模块，实现用户注册登录、发帖评论、搜索互动等基础功能。

《项目设计》：

架构/技术路线：
- 前端：HTML/CSS/JavaScript原生开发，自定义CSS组件库，响应式设计
- 后端：Django 4.2 + Django REST Framework，JWT认证机制
- 数据库：MySQL 8.0，支持关系型数据存储
- 部署：开发环境使用本地服务器，生产环境采用Nginx+Gunicorn+Django架构

模块/页面/接口清单：
1. **用户管理模块**
   - 页面：登录页、注册页、个人中心页
   - 接口：/api/auth/register、/api/auth/login、/api/auth/logout、/api/auth/profile

2. **帖子管理模块**
   - 页面：帖子详情页、发帖页面
   - 接口：/api/posts、/api/posts/{id}、/api/posts/like、/api/posts/collect

3. **评论系统模块**
   - 页面：评论区组件
   - 接口：/api/comments、/api/comments/{id}、/api/comments/like

4. **贴吧管理模块**
   - 页面：首页、贴吧列表页、贴吧详情页
   - 接口：/api/tiebas、/api/tiebas/{id}、/api/tiebas/follow

5. **搜索模块**
   - 页面：搜索结果页
   - 接口：/api/search

《任务分配》：

1. **美工/UI前端设计Agent**
   - 设计并实现7个核心页面：首页、登录/注册页、贴吧列表页、贴吧详情页、帖子详情页、用户个人中心、发帖页面
   - 制定蓝色系UI设计规范，包含CSS变量定义
   - 开发可复用组件库
   - 实现基础页面交互JavaScript逻辑
   - 确保响应式设计适配移动端、平板和桌面端

2. **后端程序员Agent**
   - 实现6个核心模块：用户认证、帖子管理、评论管理、贴吧管理、搜索模块、权限控制
   - 设计并实现4个核心数据模型：User、Tieba、Post、Comment
   - 开发20+RESTful API接口，包含完整的CRUD操作
   - 实现JWT认证和权限控制机制
   - 优化数据库查询和性能

3. **测试Agent**
   - 编写功能测试用例，覆盖用户注册/登录、帖子发布/浏览、评论发表/回复、搜索等核心功能
   - 执行性能测试，验证页面加载速度(<2秒)和API响应时间(<200ms)
   - 执行安全测试，检查SQL注入、XSS等常见漏洞
   - 生成测试报告和缺陷清单
   - 参与最终验收测试

《依赖与顺序》：

1. 后端数据库设计 → 后端模块开发
2. UI设计规范 → 前端页面实现
3. API接口定义 → 前后端并行开发
4. 后端API实现 → 前端联调测试
5. 前后端集成完成 → 测试Agent功能测试
6. 功能测试通过 → 性能优化和安全加固
7. 优化完成 → 最终验收测试

具体顺序：
- 第1-2天：后端数据库设计，前端UI规范制定
- 第3-7天：后端实现用户认证和核心API，前端实现基础页面
- 第8-14天：完成所有功能开发，开始前后端联调
- 第15-18天：全面测试和bug修复
- 第19-21天：最终验收和部署准备

《验收标准》：

1. **功能验收** - 通过黑盒测试验证各功能点是否符合需求规格
   - 通过条件：所有核心功能100%实现，无重大功能缺陷

2. **接口验收** - 使用API测试工具验证接口功能和性能
   - 通过条件：所有接口响应格式规范，状态码正确，异常处理完善

3. **性能验收** - 使用性能测试工具测量系统响应时间
   - 通过条件：页面加载时间<2秒，API响应时间<200ms，支持100并发用户

4. **安全验收** - 执行安全漏洞扫描和渗透测试
   - 通过条件：无高危安全漏洞，密码加密存储，防SQL注入和XSS攻击

5. **UI/UX验收** - 验证界面美观性和用户体验
   - 通过条件：界面符合设计规范，响应式设计正常，交互流畅自然

6. **代码质量验收** - 代码审查和静态分析
   - 通过条件：代码符合项目规范，无严重代码质量问题，文档完整
